## 🎨 5. UX 조정 (배포 후)

실제 대시보드 확인 후 사용자 피드백 반영.

### 조정 1: 폰트 크기 확대
- `tailwind.config.ts`에 `fontSize` 토큰 오버라이드: xs 12→13, sm 14→15, lg 18→20, xl 20→22, 2xl 24→28
- `src/index.css` body font-size 13→14px
- `.formula-box` text-[11px]→[12px]

### 조정 2: KPI 카드 레이아웃 6×1 → 3×2
- 기존: `lg:grid-cols-6 gap-2.5` — 카드 높이 불균일, 설명문 줄바꿈
- 변경: `lg:grid-cols-3 gap-4` — 카드 2배 넓어져 한 줄에 들어감
- 1행: 오늘 생산량 / 시간당 생산량 / 작업시간
- 2행: 불량률 / 제조원가 비율 / 고객 클레임

### 조정 3: Formula-box 폰트 통일
- 기존: `font-mono` (JetBrains Mono, 한글은 Noto Sans KR 폴백)
- 한 박스 내 한글+영문에서 두 폰트 섞임 → 시각적 어색
- 변경: `font-sans` (Noto Sans KR 단일) — 가독성 우선

### 조정 4: Mock-seed 타임존 재수정
- **증상**: 시간대별 생산량 차트 x축이 17~23시 (의도는 08~14시)
- **원인**: `(date_trunc('day', now() at time zone 'Asia/Seoul'))::timestamptz` 캐스트가 session UTC로 해석 → +9시간 밀림
- **해결**: Phase 3 타임존 fix와 동일 패턴. `fn_kst_today_start()` 헬퍼 함수로 교체.

### 조정 5: 제조원가 개선률 음수 폭발
- **증상**: mock 데이터에서 cost_ratio_pct = 100%일 때 "개선률 -1700%" 표시
- **원인**: `(15 - 100) / (15 - 10) × 100 = -1700`. 기준 초과 시 음수.
- **해결**: Clamp `Math.max(0, Math.min(100, value))` + `cost > 15` 시 "기준 초과" 라벨 + danger 색상.
- CostPage hero pill도 동일하게 수정.

### 조정 6: last_seen_at 리셋 이슈
- **증상**: 수정된 mock-seed 재주입 후에도 장비 "가동중"으로 표시
- **원인**: `fn_touch_device_last_seen` 트리거가 "앞으로만 갱신" 정책. 이전 mock의 미래 시각(KST 23:00)이 `devices.last_seen_at`에 박혀 있어 새 KST 14:00 메트릭으로 업데이트 안 됨.
- **해결**: `update devices set last_seen_at = null where code in (...)` 수동 리셋 후 mock-seed 재주입.

## 💡 오늘의 교훈

### 9.6 Vercel Hobby ≠ Private repo + Collaborator (DEV_NOTES에 추가)
- Hobby: Public OK, Private + 팀 차단
- 대안: Public / CF Pages / Vercel Pro

### 9.7 `supabase db push`는 마이그레이션만, seed.sql은 로컬 전용
- Cloud용 초기 데이터는 **마이그레이션으로 승격** 필요
- `on conflict do update/nothing`으로 idempotent 유지

### 9.8 Supabase UI는 빠르게 바뀜
- 문서에 정확한 위치보다 "어떻게 찾는지"를 써야 (URL 바 hint 등)
- 새 키 형식(`sb_publishable_`)은 UI에서 **Copy 버튼** 필수 — 드래그 선택은 축약 값 잡힘

### 9.9 트리거의 "앞으로만 갱신" 정책은 예외 상황에서 굳어짐
- 한 번 미래 timestamp가 들어가면 복구 불가
- Phase 5+ 방어 로직: bucket_at > now() + N 이면 거부

### 9.10 마크다운 렌더된 SQL 복사 주의
- invisible Unicode, smart quote 섞일 수 있음
- 파일을 `pbcopy`로 직접 복사하거나 `.sql` 파일 링크 공유가 안전
